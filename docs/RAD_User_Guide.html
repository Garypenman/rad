Analysis Workflow

From raw ROOT trees to physics histograms.

The RAD Philosophy

In modern physics analysis, a single "Event" often contains multiple "Candidates". To study a reaction like e p -> e' K+ X, you must analyze all valid combinations.

  • Define Candidates: You define what an "Electron" is (e.g., PID=11).
  • Auto-Combinatorics: RAD automatically generates the Cartesian product of all candidates.
  • Vectorized Physics: Calculations run on these combinations efficiently in a single pass.

1Setup & Beams

Initialize the reaction and define the colliding beams.

// 1. Initialize the Reaction (Data Source) epic::ePICReaction reaction("events", "data/*.root"); // 2. Setup Beams (e.g. from MC header) reaction.SetBeamsFromMC(0, 1); reaction.SetupMatching(); // Prepares Rec-to-Truth tables // 3. Register Data Tiers // Reaction handles "rec_" and "mc_" prefixes automatically

2Topology & Particles

Define Candidates (Inputs) and Composite Particles (Sum/Diff).

// --- A. Define Roles (For Truth Matching) --- const int Role_ScatEle = 2; const int Role_Proton = 3; // --- B. Select Candidates --- // "scat_ele" = All Rec tracks with PID 11 reaction.SetParticleCandidates("scat_ele", Role_ScatEle, rad::index::FilterIndices(11), {"rec_pid"}); reaction.SetParticleCandidates("p", Role_Proton, rad::index::FilterIndices(2212), {"rec_pid"}); // --- C. Configure Master Processor --- KinematicsProcessor kine(&reaction, "rec_"); // --- D. Create Composite Particles --- // J/psi -> e+ e- (Sum) kine.Creator().Sum("Jpsi", {{"ele"}, {"pos"}}); // Missing Neutron -> (Beam + Target) - (ScatEle + JPsi) kine.Creator().Diff("miss_n", {{"beam_ele", "tar_ion"}, {"scat_ele", "Jpsi"}}); // --- E. Grouping for Physics --- // Defines systems for Mandelstam calculations kine.SetBaryonParticles({"p"}); kine.SetMesonParticles({"Jpsi"});

3Physics Calculations

// --- Standard Variables --- kine.Q2(); // Creates "rec_Q2" kine.CosThetaCM(); // Creates "rec_CosThetaCM" // --- Mass --- kine.Mass("M_Jpsi", {"Jpsi"}); kine.Mass("MM_neutron", {"miss_n"}); // --- Custom Physics Kernels --- kine.RegisterCalc("t_prime", physics::TPrime);

4Cloning & Execution

The Master/Clone Workflow

Define your analysis ONCE on the Master (Rec). Clone it for Truth and other hypotheses. Initializing the Master automatically initializes the entire tree.

// 1. Stream Cloning (Rec -> Truth) // Automatically ports groups (e.g. "rec_Jpsi" -> "mc_Jpsi") auto kine_tru = kine.CloneForType(rad::consts::data_type::Truth()); // 2. Hypothesis Cloning (Linked) // Compare "Detected Proton" (Master) vs "Missing Proton" (Clone) auto kine_miss = kine.CloneLinked("_miss"); // Override: Use missing mass vector for Baryon definition kine_miss->SetBaryonParticles({"miss_n"}); kine_miss->RegisterCalc("t_alt", physics::TBot); // 3. Truth for the Missing Hypothesis auto kine_miss_tru = kine_miss->CloneForType(rad::consts::data_type::Truth()); // 4. EXECUTION (Cascade) // Calling Init() on the Master triggers all clones automatically. kine.Init(); // 5. Save reaction.Snapshot("output.root");
